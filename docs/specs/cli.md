# Spec: CLI (Textual TUI)

## Обзор

Терминальный интерфейс на Textual. Основной и единственный способ взаимодействия с игрой. Два режима запуска: игра и watch mode.

---

## Режимы запуска

### Play mode (по умолчанию)

```bash
python -m party_of_one.cli
```

Игрок видит нарратив, вводит действия, наблюдает ходы компаньонов.

### Watch mode

```bash
python -m party_of_one.cli --watch --rounds 30
```

Автопрогон без участия игрока. Нарратив выводится в реальном времени (можно наблюдать), ввод игрока заменяется на «наблюдает». По завершении — путь к логу для eval pipeline.

### Панели

| Панель | Содержание | Поведение |
|--------|-----------|-----------|
| **Нарратив** (основная) | Текст DM, реплики и действия компаньонов | Скроллится вверх, новый текст появляется снизу |
| **Партия** (боковая) | HP, STR, DEX, WIL каждого члена партии | Обновляется после каждого хода |
| **Ввод** (нижняя) | Поле ввода действия игрока | Активно только когда ход игрока |

### Визуальное оформление

- Текст DM — основной цвет
- Реплики компаньонов — выделены цветом (у каждого свой)
- Системные сообщения (сохранение, ошибки) — приглушённый цвет
- HP ниже половины — жёлтый, ниже четверти — красный
- Статус «dead» — зачёркнутый или серый

## Команды игрока

| Команда | Действие |
|---------|---------|
| Свободный текст | Действие персонажа (передаётся DM) |
| `/save` | Сохранить сессию |
| `/quit` | Сохранить и выйти |
| `/status` | Показать полное состояние партии и квестов |
| `/help` | Список команд |

Команды начинаются с `/`, всё остальное — игровое действие.

## Поток взаимодействия

```
1. Запуск:
   ├── Есть сохранённая сессия → «Продолжить / Новая игра» (стрелки + Enter)
   └── Нет сохранённой сессии → экран создания игры

2. Экран создания игры (если новая)
   ├── Выбор архетипа персонажа (стрелки + Enter)
   ├── Выбор двух компаньонов из списка (стрелки + Enter)
   └── Описание сеттинга (свободный текст)

3. Инициализация → DM генерирует начальную сцену
   └── Нарратив появляется в основной панели

4. Игровой цикл:
   ├── Поле ввода активно → игрок вводит действие → Enter
   ├── Поле ввода заблокировано → DM обрабатывает ход
   ├── Нарратив DM появляется в основной панели
   ├── Компаньон A: действие + реплика
   ├── DM обрабатывает ход компаньона A
   ├── Компаньон B: действие + реплика
   ├── DM обрабатывает ход компаньона B
   └── Поле ввода снова активно

5. Завершение → финальный нарратив DM → экран завершения
```

### Индикация ожидания

Пока DM или компаньон думают — анимация в поле ввода (spinner или «DM думает...»). Игрок видит, что система работает, а не зависла.

---

## Экран создания игры

Последовательный выбор с помощью стрелок:

```
Выберите архетип персонажа:
  > Воин
    Следопыт
    Маг
    Плут
```

После выбора архетипа — аналогичный список компаньонов (выбрать двух). Затем — поле для свободного описания сеттинга.

---

## Watch mode (отличия от play)

- Поле ввода отсутствует
- Вместо действия игрока отображается «*Игрок наблюдает*»
- В header — счётчик раундов и прогресс
- По завершении — summary: сколько раундов, чем закончилось, путь к логу
